# Final Reflection

## What I Learned About Backend Development

Throughout this project, I learned how backend systems are structured using Node.js, Express, and MySQL. I gained experience creating API routes, connecting a backend application to a database, handling requests and responses, and organizing a project into middleware, routes, and utility files. I also learned how authentication and authorization work using JWT tokens and role-based middleware.

## What I Learned About Working With a Shared Codebase

Working on a shared codebase taught me the importance of organization, naming consistency, and version control. I learned how small mistakes in file structure or route imports can affect the entire project. Using GitHub commits helped track changes and made it easier to debug problems throughout development.

## One Technical Challenge I Solved

One major challenge I solved involved fixing route and middleware structure problems that caused module import errors. Some files were placed in incorrect folders, which caused the application to fail when starting the server. I corrected the folder structure, updated imports, fixed middleware placement, and successfully restored the application so it could run correctly again.

## One Area I Would Improve With More Time

If I had more time, I would improve the frontend integration and add more advanced validation and testing. I would also improve the overall API documentation and create a more polished user interface connected to the backend system.

## How This Project Represents My Skills Professionally

This project demonstrates my understanding of backend development fundamentals, including REST APIs, database integration, middleware usage, authentication, error handling, and project organization. It also reflects my ability to troubleshoot technical problems, improve code structure, and manage a full backend application using GitHub and professional development tools.
