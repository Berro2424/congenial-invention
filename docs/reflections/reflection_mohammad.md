

# Final Reflection – Mohammad berro

This backend project taught me a lot about how real backend systems are built and managed. Throughout the semester, I worked with Node.js, Express, and MySQL to create a working API that handled users, authentication, reservations, validation, and database communication. In the beginning, I was mainly focused on just getting the routes to work, but over time I started understanding how important organization, middleware, debugging, and clean code structure really are.

One of the biggest things I learned from this project was how much problem solving goes into backend development. There were many times where the smallest mistake, like a wrong file path or misplaced folder, could stop the entire server from running. At first those errors were frustrating, but after working through them I became much more comfortable reading terminal errors, debugging issues, and figuring out how everything in the project connects together.

I also learned how important it is to keep a project organized when multiple files and features are involved. As the API grew larger, separating routes, middleware, utility functions, and database logic made the project much easier to manage. I realized that writing cleaner and more readable code not only helps other people understand the project, but also makes it easier for me to fix issues later on.

One technical challenge I dealt with was reorganizing the project structure and fixing module path errors. After moving files into different folders, the server stopped working because some import paths were incorrect. I had to go through the project carefully, fix the paths, and reorganize the folders until everything worked again. That experience helped me better understand how backend applications are structured and connected behind the scenes.

If I had more time to continue working on the project, I would improve validation, add stronger security features, and create better API documentation. I would also like to improve the frontend integration so the project feels more complete overall.

Overall, this project helped me grow a lot as a developer. It gave me real experience building a backend system, troubleshooting problems, working with databases, and organizing a larger codebase. I feel much more confident now than I did at the beginning of the course, and I think this project shows the progress I made throughout the semester.
